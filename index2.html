    network: 'mainnet-beta',
    wallet: yourWallet
});

    name: 'YourTibiaChar',
    world: 'Antica'
});

// Perform a swap
await exura.swap({
    from: 'SOL',
    to: 'TIBIA_COINS',
    amount: 100,
    character: character.name
});

Glossary

Essential terminology for understanding blockchain gaming and asset trading.

Blockchain Basics

Blockchain

A decentralized, digital ledger that records transactions across a network of computers. It's the technology that powers cryptocurrencies and digital assets.

Smart Contract

Self-executing programs stored on a blockchain that automatically process transactions when predetermined conditions are met.

SPL Token

Solana Program Library Token - the standard for creating digital assets on the Solana blockchain, similar to ERC-20 tokens on Ethereum.

Asset Trading

Liquidity Pool

A collection of funds locked in a smart contract that enables trading between different assets. Users can contribute to these pools to earn fees from trades.

Slippage

The difference between the expected price of a trade and the price when the trade executes, often due to market movement or large trade sizes.

Asset Swapping

The process of exchanging one digital asset for another at a predetermined rate, executed through smart contracts.

Wrapped Assets

Asset Wrapping

The process of converting a game asset into a blockchain token that represents its value and can be traded on decentralized exchanges.

Backing Ratio

The relationship between wrapped tokens and their underlying assets (e.g., 1:1 means one wrapped token equals one game asset).

Custodial Service

A system that securely holds game assets while their wrapped versions are being traded on the blockchain.

Platform Features

DeFi Integration

The ability to use wrapped game assets in decentralized finance protocols, enabling features like lending, yield farming, and liquidity provision with gaming assets.

Oracle

External data feeds that provide real-time information (like prices or game data) to smart contracts.

Gas Fees

Transaction costs paid to the network for processing blockchain operations.

Multi-sig

Security feature requiring multiple approvals before a transaction can be executed.
